======================================================================
1. TEKNOLOGI & ARSITEKTUR SISTEM (DECOUPLED ARCHITECTURE)
- Backend Server: Python murni menggunakan framework FastAPI. Library pengolahan data wajib menggunakan Pandas, NumPy, Scipy, dan Openpyxl (untuk pembacaan .xlsx).
- Frontend Client: Next.js (App Router, TypeScript, Tailwind CSS). UI menggunakan komponen Shadcn UI, dan grafik interaktif menggunakan Shadcn Chart (wrapper Recharts).
- Aturan Komunikasi Data: Terpisah (Decoupled). Backend Python tidak boleh merender HTML atau file gambar statis (.png/.jpg). Backend murni bertindak sebagai REST API yang menerima file via multipart/form-data dan mengembalikan seluruh hasil kalkulasi dalam format JSON murni dengan format camelCase (contoh: `rowCount`, `dataType`). Next.js bertugas mengambil JSON tersebut dan merendernya secara interaktif di browser.

======================================================================
2. MINI-PRD (PRODUCT REQUIREMENT DOCUMENT) & TARGET FASE 1
- Fitur Utama: Upload file (.csv, .xlsx, .xls, .txt) maks 10MB ➔ Deteksi tipe data otomatis per kolom ('Numerical', 'Categorical', 'Datetime') ➔ Preview 10 baris pertama data ➔ Kalkulasi statistik deskriptif dasar (Numerik: Mean, Median, Min, Max, Std Dev; Kategorikal: Unique Values, Mode, Frequency).
- Kasus Batas (Edge Cases): File kosong/rusak menghasilkan HTTP Exception 400. Baris data yang kosong (NaN/Null) harus diabaikan saat inferensi tipe data agar tidak merusak deteksi otomatis.
- Cakupan: Sistem pada Fase 1 ini bersifat READ-ONLY (tidak ada manipulasi data atau data cleaning).
